Rubric1: The submission includes all 9 required PixelRift website areas.

Rubric2: The Home Page clearly presents PixelRift as an online gaming website.

Rubric3: The Home Page includes logo, navigation, hero section, action buttons, featured games, tournament preview, top players preview, and footer.

Rubric4: The navigation includes Home, Games, Tournaments, Leaderboard, Profile, Contact, and Login or Logout.

Rubric5: The navigation works correctly on desktop, tablet, and mobile.

Rubric6: The Register page includes name, email, username, password, confirm password, and register button.

Rubric7: The Register page shows an error for empty required fields.

Rubric8: The Register page shows an error when password and confirm password do not match.

Rubric9: The Register page shows a success message after valid registration.

Rubric10: The Login page includes email or username, password, login button, and register link.

Rubric11: The Login page shows an error for wrong login details.

Rubric12: The login or logout state updates the navigation correctly.

Rubric13: The Games Listing Page shows at least 12 original sample games.

Rubric14: Each game card shows image, name, game type, short description, rating, Play button, and View Details button.

Rubric15: The game names are original and do not copy real game titles or brand names.

Rubric16: The Games Listing Page includes search, filtering, or another clear way to browse games.

Rubric17: Each Game Details Page shows title, image or banner, game type, description, rating, difficulty, player count, how to play, Play Now button, and related games.

Rubric18: The Play Now button opens the Game Play Page correctly.

Rubric19: The Game Play Page includes one real playable browser game.

Rubric20: The playable game includes start, pause, score, timer or progress, and back button.

Rubric21: The playable game responds to user action and updates score or progress.

Rubric22: The Tournament Page shows at least 5 sample tournaments.

Rubric23: Each tournament shows name, game, date, time, player count, status, and Join button.

Rubric24: A logged in user can join a tournament and see a success or joined message.

Rubric25: A logged out user sees a login required message when trying to join a tournament.

Rubric26: The Leaderboard Page shows at least 15 sample players with rank, username, game, score, wins, and points.

Rubric27: The Leaderboard Page clearly highlights the top 3 players.

Rubric28: The Profile Page shows user details, game totals, joined tournaments, favorite games, and edit profile option.

Rubric29: The Contact Page validates required fields and shows a success message after valid submission.

Rubric30: The final website runs locally, includes clear README setup steps, uses safe assets only, stays responsive, and avoids restricted or copied content.
